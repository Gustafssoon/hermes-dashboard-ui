# Pantheon — Data Contract

This is the single seam between the mock prototype and a future real,
**read-only** status feed. Today every value lives in
[`data.jsx`](./data.jsx) as a plain object on `window.PANTHEON_DATA`. To wire
real data later, produce the same shapes — no component changes required.

> Posture: **read-only**. The portal observes and guides. Nothing in this
> contract describes a write/control action, and none should be added.

## States enum
Every status field uses one of:

```
"nominal" | "attention" | "degraded" | "down" | "info"
```

These map directly to CSS classes in `kit.css`
(`s-<state>` for dots, `b-<state>` for badges), so a collector only needs to
emit the string.

## Top-level shape

| Key | Type | Feeds | Suggested real source |
| --- | --- | --- | --- |
| `meta` | object | topbar, footer, poll interval | collector heartbeat |
| `overview` | object | Athena hero + counts | derived from `services` |
| `services[]` | array | Athena service list, incident badges | per-probe rollup |
| `nas` | object | Hephaestus → Storage | Synology DSM API (read-only) |
| `dns` | object | Oracle → DNS resolution | Technitium stats API |
| `hermesAgent` | object | Hermes → agent card | Hermes `/api/status` |
| `links[]` | array | Hermes → quick links | static config |
| `routing` | object | Hermes → remote access | Twingate connector status |
| `infra` | object | Hephaestus → host + stacks | Portainer / Docker API |
| `checks[]` | array | Oracle + Athena signals | the read-only probe runner |
| `certificates[]` | array | Oracle → TLS certificates | cert expiry scanner |
| `backups[]` | array | Aegis → backup jobs | Hyper Backup / job logs |
| `restore` | object | Aegis → restore verification | restore-drill log |
| `hue` | object | Hue service detail | Hue Bridge API |
| `runbooks[]` | array | Scrolls | static markdown/config |
| `incident` | object | Incident mode overlay | static config |

## Field reference (key objects)

```jsonc
// services[]
{
  "id": "nas",                  // stable key
  "name": "Synology DSM",
  "kind": "NAS",                // short category chip
  "icon": "hard-drive",         // Lucide name (see icons.jsx)
  "state": "nominal",
  "detail": "DS225 · DSM · 41°C",
  "checked": "3m ago",          // relative freshness
  "host": "nas.tingzel.lan:5001",
  "metrics": [["Temp", "41°C"]] // compact [label, value] pairs
}

// nas
{ "model", "dsm", "temp", "fan", "uptime", "state",
  "volumes": [{ "name", "raid", "used" /* 0-100 */, "total", "smart", "state", "note" }],
  "disks":   [{ "bay", "model", "smart", "temp", "hours" }] }

// dns
{ "service", "state", "uptime", "queriesPerMin", "blockedPct", "cacheHitPct",
  "avgLatencyMs", "upstreams": [{ "name", "addr", "proto", "latency", "state" }] }

// hermesAgent  (mirrors the real Hermes /api/status fields)
{ "version", "release", "sessions", "gatewayState",
  "configVersion", "authRequired", "state" }

// routing  (Twingate — status only, never changes routing)
{ "connector", "state", "devices", "network", "lastHandshake", "note" }

// infra
{ "host": { "name", "os", "uptime", "cpu", "mem", "disk" /* %s */ },
  "endpoint": { "name", "state", "docker", "containers", "running" },
  "stacks": [{ "name", "containers", "state", "note" }] }

// checks[]
{ "name", "result", "value", "state", "ago" }

// certificates[]
{ "host", "issuer", "days" /* until expiry */, "state", "note" }

// backups[] + restore
{ "name", "target", "size", "last", "state", "verified" }
{ "lastDrill", "target", "outcome", "cadence", "state", "note" }
```

## Mapping to the Hermes API (existing source)
The `hermesAgent` object is intentionally aligned with the real Hermes
dashboard API documented in
[`Gustafssoon/hermes-dashboard-ui`](https://github.com/Gustafssoon/hermes-dashboard-ui):
`version`, `release_date` → `release`, `active_sessions` → `sessions`,
`gateway_state` → `gatewayState`, `config_version` → `configVersion`,
`auth_required` → `authRequired`. A thin adapter that renames those fields is
all the Hermes section needs.

## How to swap in real data later (frontend-only)
1. Replace the literal in `data.jsx` with a `fetch()` of a **read-only** JSON
   endpoint that returns this shape (e.g. an nginx-served static `status.json`
   regenerated by a collector, mirroring how the Hermes frontend polls its API).
2. Keep `window.PANTHEON_DATA` as the assignment target, or lift it into React
   state and re-render on a `meta.pollSeconds` interval.
3. Do **not** introduce write endpoints, auth, or secrets in the frontend.
